## Deviations from the approved plan

1. **Bug found in Phase 0 `Ecdh1PuKdf.SuppPubInfo` layout.** The original wrapper omitted
   the 4-byte big-endian length prefix on the AEAD tag inside SuppPubInfo. Plan §3.7
   anticipated KAT mismatches as the discovery vehicle; that's exactly what happened —
   all SICPA C.3 authcrypt vectors failed AES-KW unwrap with "integrity check failed".
   Fix applied + the Phase 0 differential test was updated to match the corrected
   layout (matches askar-crypto, which produced the SICPA vectors).
2. **`apu` encoding correction.** Originally `apuBytes = Encoding.ASCII.GetBytes(apuB64u)`
   (the ASCII bytes of the base64url string). draft-madden-04 §2.3 + the askar source
   require the **base64url-DECODED** value (raw UTF-8 of `skid`) as `PartyUInfo`. Both
   the pack and unpack paths now use `Encoding.UTF8.GetBytes(skid)` / `Base64Url.Decode(header.Apu)`.
3. **`UnsafeRelaxedJsonEscaping` in `DidCommJson` + `DeterministicJsonWriter`.** The
   default `System.Text.Json` encoder escapes `+` → `\u002B`. Spec vectors carry the
   literal character; deterministic JSON bytes had to match.
4. **`AnonymousSender` semantics decoupled from `Authenticated`.** Phase 2 first tried
   `anonymous = anonymous && !authenticated` which was wrong for `anoncrypt(sign())` —
   per the SICPA reference, the two flags are independent (anonymous reflects only the
   outermost protective layer; authenticated reflects whether *any* layer bound a
   sender identity). Updated `EnvelopeReader` + the test expectation.
5. **JWS `kid` in protected OR unprotected header.** Originally the parser required the
   protected header to carry `kid`; spec vectors carry it only in the unprotected
   header. Now both are accepted; when both are present they MUST match.
6. **Phase 2 ships internal contracts**, not the public `ISecretsResolver` /
   `IDidKeyService`. That keeps Phase 3's facade decisions unblocked; the only consumers
   today are envelope code + tests.
7. **Outbound interop deferred.** Manifest dispatcher recognizes `pack-*` operations
   but short-circuits — outbound bytes are non-deterministic for anon/authcrypt and the
   FR-IX-03 outbound row gates on the SICPA demo CLI (Phase 6). All eight spec
   inbound vectors pass today; that's the interop gate the PRD calls for.

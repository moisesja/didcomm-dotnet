# PR #7 review remediation — address all Phase 5 transport findings

Scope: fix the 11 findings raised in the code review of PR #7 (Phase 5 transports).
Branch: `claude/romantic-ptolemy-85bffc`. No PRD/spec change required — these are
implementation-correctness and API-hygiene fixes within the shipped surface.

## Findings → fixes

### Notable

**N1 — WebSocket failures must surface as `TransportException` (FR-API-07).**
`WebSocketDidCommTransport.SendAsync` lets the raw `WebSocketException` /
`TimeoutException` / `OperationCanceledException` escape after the reconnect
budget is exhausted, violating the `IDidCommTransport` / `TransportException`
contract (HTTP wraps; WS doesn't).
- Add a ctor to `TransportException`: `(string message, Exception innerException, int? httpStatusCode, string? scheme)` so WS can carry both inner + scheme.
- Wrap the `_reconnectPipeline.ExecuteAsync(...)` call in `SendAsync`:
  - rethrow `OperationCanceledException when (ct.IsCancellationRequested)` (caller cancel ≠ transport failure),
  - rethrow existing `TransportException`,
  - wrap any other exception in `TransportException(..., ex, null, request.Endpoint.Scheme)`.

**N2 — Socket leak on connect failure.**
In `GetOrConnectAsync`, the `factory()` socket isn't disposed if `connect(...)`
throws (refused / `ConnectTimeout`). Wrap connect in try/catch that disposes the
nascent socket before rethrowing. (Amplified by reconnect retries.)

**N3 — Malformed `Content-Type` → unhandled `FormatException` → HTTP 500.**
`MatchesMediaType` calls `new ContentType(contentType)`, which throws on a
malformed header value; any unauthenticated client can drive a 500.
- Catch `FormatException` inside `MatchesMediaType` → return `false` (→ 415).
- Tighten the endpoint guard to `string.IsNullOrEmpty(contentType)`.

### Minor

**M4 — Dead `WebSocketTransportOptions.Clock`.** Declared, never read (grep
confirms zero usages). Delete the property + its doc comment.

**M5 — `Disconnected` / `Reconnected` lifecycle events never fire.** Only
`Connected` + `SendFailed` are raised today.
- Move socket acquisition inside the send try so a connect failure also raises `SendFailed`.
- On a failed attempt where a previously-pooled socket was dropped → raise `Disconnected`.
- On a send that succeeds on a retry attempt (`attempt > 0`) → raise `Reconnected`.
- On `DisposeAsync` per-socket close → raise `Disconnected` (endpoint reconstructed from the pool key, which is a well-formed absolute URI).

**M6 — `CircuitBreakerFailureThreshold` < 2 throws Polly `ValidationException`.**
Polly's `MinimumThroughput` is `[Range(2, …)]`. Clamp
`MinimumThroughput = Math.Max(2, options.CircuitBreakerFailureThreshold)` in
`HttpResiliencePipelineFactory` + note the floor on the option's XML doc.

**M7 — Single global `_connectLock` serializes connects across all endpoints.**
Replace the single `SemaphoreSlim` with a per-pool-key
`ConcurrentDictionary<string, SemaphoreSlim>`; dispose all gates in `DisposeAsync`.

**M8 — DRY: `DidCommReceiveOptions.AcceptedMediaTypes` hardcodes the media-type
strings.** Reference the existing `DidCommMediaTypes.{Encrypted,Signed,Plain}`
constants (same assembly).

**M9 — `Task.Delay(50)` drains in WS tests + cookbook (flaky).** Replace the
fixed sleeps with a bounded poll on the `received` collection
(`WaitUntilAsync(() => received.Count >= n, 5s)`); pass `received` into the
WS-send helper so it polls before disposing the transport. 3 sites:
`WebSocketTransportRoundTripTests.cs:85,151`, `Section_R_ReceiveWebSocket.cs:76`.

**M10 — Dead `catch (TransportException) → 502` in `MapDidCommEndpoint`.**
`UnpackAsync` can't throw it. Remove the catch (XML doc already lists only
202/415/413/400/500).

**M11 — `(WebSocketCloseStatus)1009` magic cast.** Replace with
`WebSocketCloseStatus.MessageTooBig` (== 1009) in `ReceiveLoopAsync`.

- **N1** `TransportException` gained a `(message, inner, httpStatusCode, scheme)` ctor; `WebSocketDidCommTransport.SendAsync` wraps the exhausted pipeline (preserving caller cancellation + not double-wrapping existing `TransportException`).
- **N2** `GetOrConnectAsync` disposes the nascent socket on connect failure/timeout.
- **N3** `MatchesMediaType` catches `FormatException` → 415; endpoint guard tightened to `IsNullOrEmpty`.

- **M5** `Disconnected` (drop + dispose) and `Reconnected` (retried success) now fire; connect failures count as `SendFailed`.
- **M6** `MinimumThroughput = Math.Max(2, threshold)` + option doc note.
- **M7** per-pool-key `SemaphoreSlim` connect gates; disposed in `DisposeAsync`.
- **M8** `DidCommReceiveOptions` reuses `DidCommMediaTypes` constants.
- **M9** WS round-trip tests + Cookbook R use bounded polling instead of `Task.Delay(50)`.
- **M10** removed dead `TransportException → 502` catch.
- **M11** `WebSocketCloseStatus.MessageTooBig` replaces the `(WebSocketCloseStatus)1009` cast.

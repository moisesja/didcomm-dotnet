# PR #51 root-cause remediation plan

Target: `feat/discover-features-initiator-49` at
`5c50b928bf3f67553a9d6866cd063efd58fa3517`.

Objective: end the repeated PR-review cycle by replacing the remaining local guards with one
trust-boundary/lifecycle design, fixing every open review finding, preserving the 1.2 public ABI,
and proving the fixes under real cryptography, concurrency, mutation, overflow, and shutdown.

This plan deliberately does not modify source, tests, product docs, or GitHub state until the user
approves it. The existing `.claude/settings.json` change and untracked task files are user-owned and
will not be touched.

## Non-negotiable invariants

1. **Cryptographic metadata is the authority for identities.** The authenticated peer comes from a
   verified `SenderKid`/`SignerKid`; the local identity comes from the `RecipientKid` that actually
   decrypted. Handler-controlled `message.To[0]`, `message.From`, and `reply.From` are never authority
   for automatic egress.
2. **One immutable inbound truth is born at successful unpack.** Correlators and observers consume the
   exact verified plaintext plus copied envelope metadata, never a mutable `Message` after a handler
   has seen it and never N independently timed snapshots.
3. **Admission happens before materialization.** An observer consumes one item slot and its exact UTF-8
   byte charge before any observer-specific deserialization. A full queue performs zero clone work.
4. **Correlation claims, parsing consumes.** The receive path performs only immutable header/trust
   checks and an atomic pending-query transition before arbitrary handlers. Disclosure parsing runs
   on the awaiting initiator continuation, not on the receive path and not on an unbounded `Task.Run`
   queue.
5. **Shutdown semantics are honest and bounded.** Admission stops atomically, queued plaintext is
   drained/released, cooperative callbacks are cancelled, tasks and faults are observed, and the CTS
   outlives every pump. A callback that ignores cancellation cannot be killed by .NET; at most that
   single in-flight observation may remain until trusted observer code returns.
6. **No public compatibility churn.** Preserve every shipped public constructor, method, positional
   record shape, and `AllowSameSocketReplies`. The security fix is a fail-closed behavioral tightening
   behind the existing opt-in WebSocket option.

## Implementation

### 1. Freeze verified inbound data once at the cryptographic boundary

- [x] Add an internal immutable `InboundMessageSnapshot` containing the exact innermost plaintext JSON,
  exact UTF-8 byte count, copied `id`/`type`/`thid`/`from`/`to`, and copied trust metadata including
  sender/signer/recipient kids.
- [x] Register the snapshot in `EnvelopeReader` only after message validation plus all envelope and
  resolver-backed consistency checks succeed. Reuse the existing `current` plaintext string: no
  serialize/deserialize pass is added to normal unpack.
- [x] Carry the snapshot through a `ConditionalWeakTable<Message, InboundMessageSnapshot>` whose value
  never references its key. This leaves public `Facade.UnpackResult` constructor/deconstruction,
  equality, hash, and `ToString` untouched.
- [x] For host-constructed `UnpackResult` instances, provide one guarded compatibility fallback that
  serializes once before any handler when a correlator/observer actually exists. Snapshot failure is
  logged and skips side-channel work; it never changes handler dispatch or its outcome.
- [x] Copy every mutable collection used for trust decisions. No snapshot field may expose mutable
  `Message`, `JsonNode`, or caller-owned list state.

### 2. Make Discover Features correlation lossless with respect to observer queues and cheap on receive

- [x] Change the internal-only `IInboundCorrelator` contract to consume `InboundMessageSnapshot`.
- [x] In `ProtocolDispatcher`, obtain/guard the snapshot and invoke correlators before
  `DispatchCoreAsync`, so handler delay, mutation, cancellation, or exceptions cannot suppress or
  forge correlation. Keep observer delivery best-effort and independent of the outcome.
- [x] Change each Discover Features pending entry to store both requester and responder DID subjects
  plus a `TaskCompletionSource<InboundMessageSnapshot>` created with
  `RunContinuationsAsynchronously`.
- [x] Validate immutable type/thread, authenticated key presence, key-to-`from` subject binding,
  responder subject, signed `to` recipient, and—for encrypted replies—`RecipientKid` to requester
  subject before claiming a response. A response for another hosted local identity must not complete
  the waiter.
- [x] Make response, timeout/caller cancellation, and cleanup compete through one atomic pending-entry
  removal/terminal transition. A duplicate response does no parsing; a rejected forgery leaves the
  legitimate pending entry intact.
- [x] Have `QueryFeaturesAsync` deserialize/read disclosures only after awaiting the winning immutable
  snapshot, with cancellation/deadline checks around parsing. Keep transport failures unchanged and
  keep the existing whole-operation timeout contract.
- [x] Replace per-packet warning amplification for known-thread forgeries with a first-plus-periodic
  structured summary (or debug detail), and correct all “O(1)/lossless/non-blocking” claims to the
  precise guarantee: the inline matching/claim phase is constant-time and is not subject to observer
  queue drops; parsing is off the receive path.

### 3. Rebuild observer delivery around admission-before-work and an explicit lifecycle

- [x] Queue the shared immutable snapshot, not a prebuilt `InboundObservation` and not the live
  `UnpackResult`.
- [x] Store each configured capacity. Under a short per-observer lock, atomically reserve one
  outstanding item and the exact snapshot UTF-8 byte charge, reject oversize/count-full/byte-full
  work, and roll back both reservations on every `TryWrite`/shutdown failure.
- [x] Count queued plus in-flight work in the hard bounds. Release exactly once in a pump `finally` or
  disposal drain, making the maximum retained library-owned work explicit and testable.
- [x] Deserialize a private `Message` and build `InboundObservation` inside each accepted observer's
  pump. Catch materialization and callback failures there so neither can clobber dispatch and two
  observers can never share mutable message state.
- [x] Fix overflow logging so `{MessageId}`, the actual configured `{Capacity}`, `{ByteBudget}`, and
  `{Dropped}` receive the correct structured values; preserve rate limiting.
- [x] Reject enqueue/flush after shutdown before any materialization. Capture the shutdown token before
  starting pump tasks.
- [x] Replace the single `_disposed` early-return behavior with idempotent `BeginShutdown`: stop
  admission, complete writers, cancel pumps, concurrently drain queued items/barriers and release
  reservations, and expose one observed pump-completion task. `Dispose()` starts shutdown without
  blocking; `DisposeAsync()` waits the documented grace even if sync disposal already started.
- [x] Dispose the CTS only after all pumps eventually exit. A genuinely non-cooperative in-flight
  callback may outlive the grace period, but it cannot retain the rest of the queue or touch a
  prematurely disposed CTS.

### 4. Bind every automatic same-socket reply to the actual cryptographic participants

- [x] Preserve `DidCommReceiveOptions.AllowSameSocketReplies`, but make the internal route gate require
  an authenticated encrypted inbound, a parseable authenticating sender/signer kid, and a parseable

  selected local recipient identity.
- [x] Derive the peer DID subject from `SenderKid`/`SignerKid` and the local DID subject from

  assembly rather than duplicating DID parsing).
- [x] Require `reply.From` to be the local recipient subject and require all `reply.To` entries to name

- [x] Deep-snapshot the handler reply once before validation and packing, closing mutation between the

- [x] Update built-in Discover Features and Trust Ping handlers to prefer the decrypting recipient DID

- [x] Correct the two-agent HTTP responder callback to derive/validate both directions from key IDs,
  then send to the authenticated peer as the decrypting local identity. Never pass handler-selected
  `replyFrom` directly as `SendOptions.From`.

### Adversarial review

The independent audit found four concrete issues and one explicit tradeoff:

1. Reentrant fast sends could parse a response on the inbound transport stack — fixed with an
   unconditional forced asynchronous continuation and a dedicated thread-identity regression.
2. Sender/signer key presence was not enough; their DID subjects were not bound to plaintext `from`
   — fixed and tested for both sender and signer mismatches.
3. Synthetic fallback could serialize one mutable state then reread headers from another — fixed by
   deriving all message headers/recipients from the serialized clone.
4. Cancellation could contradict a response that already won terminal completion — fixed with one
   pending-state CAS/TCS settlement and a 256-iteration barrier race test.
5. The weak sidecar necessarily retains the raw plaintext while the corresponding public `Message`
   remains live. This is the deliberate cost of preserving unpack-time truth without changing the
   shipped `UnpackResult` ABI; the value never references its weak key and is collected with it.

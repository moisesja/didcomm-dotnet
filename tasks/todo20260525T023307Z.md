### Clear, low-ambiguity fixes
- [ ] **#2 base64url decode** (`ForwardProcessor.cs:116`): replace
  `Convert.FromBase64String(attachment.Data.Base64)` with `Base64Url.Decode(...)`
  (the repo helper used everywhere else). Add a test: forward whose payload is a
  base64url `data.base64` round-trips through the processor.
- [ ] **#3 delay overflow** (`ForwardProcessor.cs:132-133`): guard the negative branch so
  `long.MinValue` and values whose `|n|` exceeds `int.MaxValue` can't overflow
  `Math.Abs`/`bound+1`/`(int)`. Clamp the upper bound to `int.MaxValue` safely. Add tests
  for `long.MinValue` and `-(long.MaxValue)`.
- [ ] **#4 TryParse non-string `next`** (`ForwardMessage.cs:113`): inspect the JSON token
  kind before reading; throw `MalformedMessageException` (not `InvalidOperationException`)
  when `next` is present but not a string. Add a test.
- [ ] **#6 relative-fragment routing key** (`MediatorEndpointExpander.cs:114-118`): treat
  `hashIndex <= 0` (empty subject DID) as a malformed routing key and throw
  `DidResolutionException` with a clear message instead of a bare `ArgumentException`.
  Add a test.
- [ ] **#8 DI registration** (`DidCommServiceCollectionExtensions.cs:42`): change
  `AddSingleton` → `TryAddSingleton` for `DidCommClient` for idempotency parity with the
  sibling registrations. Add a DI test asserting a single registration after double-add.

### Fixes needing a design decision (see questions)
- [ ] **#1 rewrap self-reference** (`ForwardProcessor.cs:145`): rewrapped forward sets
  `next == to == nextHop`. The spec leaves rewrapping partly implementation-defined and the
  mediator cannot know the hop beyond `next` (it is inside the inner, encrypted attachment).
  Options: (A) treat `next == self` as the intended "final hop for this layer" signal and
  add a forward-aware round-trip test + docs; (B) keep behavior but mark rewrap mode
  experimental and document the forward-aware-recipient requirement; (C) defer — leave code
  as-is, downgrade the finding, fix only the multi-hop fragment'd-`next` feeding
  `GetVerificationMethodsAsync`.
- [ ] **#5 multi-attachment dropped** (`ForwardProcessor.cs:91`): processor relays only
  `attachments[0]`. Options: (A) fail fast — throw if >1 attachment (Phase 4 = single
  payload); (B) relay all (changes `ForwardProcessingResult.OnwardPacked` to a list — larger
  surface change).
- [ ] **#7 fallback URIs not DID-expanded** (`MediatorEndpointExpander.cs:56`): Options:
  (A) drop DID-valued fallbacks (only surface real transport URIs); (B) fully expand each
  fallback (more work, recursion guard); (C) leave as-is (Phase 5 concern) — document.
